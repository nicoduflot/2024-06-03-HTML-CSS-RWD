Les images

Les images peuvent être affichées à l'aide de deux balise : img et picture

img

img est une balise qui une fois écrite, va aller chercher l'image source et l'afficher à sa place. Si rien d'autre n'est précisé, l'image s'affiche à sa taille originale. Si on change, par exemple, sa largeur, la hauteur qui le ratio (l'homothétie, le rapport hauter / largeur)

img peut être appelé dans et hors paragraphe.

picture

La balise picture permet de faire de l'affichage d'image adaptatif : selon la taille disponible autour de l'image afficher, on peut sélectionner la même image avec des formats différents.

Figure

Il existe une balise conteneur pour les media, qui aide la sémantique web pour le référencement et l'accessibilité. la balise figure va encadrer un media, et donne la possibilité d'y ajouter une légende, avec la balise figcaption.

La légende sera rattachée sémantiquement à l'image et son texte alternatif, ce qui améliorera le référencement.

L'audio

L'audio hébergé sur le site

Pour mettre à disposition un fichier audio sur un site, on utlise la balise audio. Cette balise peut soit appeler un seul fichier, ou utliser aussi la balise source, déjà vu associée à la balise picture. Utiliser plusieurs source permet de s'adapter à la configuration de l'utilisateur. Un appareil, pour lire un fichier audio (ou vidéo) doit posséder les bon codecs. si je met a disposition un fichier avec une extension que l'appareil qui essai de le jouer ne connaît, j'aurais une erreur. Si je propose le même son dans des format différent avec source, l'appareil lancera le premier fichier qu'il est capable de lire.

L'audio hébergé sur un site tiers

Les média audio (et aussi les vidéos) hébergés sur des site tiers, quand elles partageable, le sont grâce au princope des iframes. L'iframe est un cadre quio permet d'appeler une autre ressource, comme par exemple une page web.

Comme l'iframe appel un page html, cette page contient le fichier audio et un lecteur, dans même esprit que celui qu'on a créé précédement.

Les vidéos hébergées sur un site tiers

Youtube

Les vidéo hébergées posent souvent une défi : les tailles de l'iframe sont bloquée pour pouvoir profiter du ratio de la vidéo (16 / 9, ou du 4 / 3). Il faut créer des classes CSS particulières pour pouvoir ajouter des vidéos via iframe qui gardent leur ratio tout en étant adaptatives.
